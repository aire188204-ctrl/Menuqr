================================================================================
KEY FEATURES IMPLEMENTED
================================================================================

Race Condition Prevention:
- Two cashiers scanning same device both see "Available"
- First cashier locks row with SELECT...FOR UPDATE
- Second cashier waits, gets 409 Conflict when row becomes Sold
- Clear error message: "Device sold by another cashier"

Atomic Transactions:
- Create sale + sale_items + warranties + movements in single transaction
- All succeed or all rollback (no partial sales)
- Guarantees data consistency

Warranty Auto-Generation:
- Start date: Sale date
- End date: Sale date + 12 months
- Type: "Standard"
- Status: Active

Performance Optimization:
- Index on imei_1 (IMEI scan < 100ms)
- Index on serial_number (Serial scan < 100ms)
- Index on status (Inventory queries fast)
- Composite indexes for multi-column queries
- Connection pooling (20 max concurrent)

Multi-Tenant Support:
- tenant_id on every table
- Complete data isolation
- Support for multiple stores/branches

Error Handling:
- DEVICE_NOT_FOUND (404) - IMEI not in system
- DEVICE_NOT_AVAILABLE (400) - Already sold/defective/reserved
- CONCURRENT_SALE_CONFLICT (409) - Sold during checkout
- INVALID_PAYMENT_METHOD (400) - Unsupported payment type
- INVALID_IMEI (404) - IMEI UUID doesn't exist
- INTERNAL_SERVER_ERROR (500) - Database/server error

SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

1. "Cannot connect to database"
   → Check DATABASE_URL is correct
   → Verify Neon project is running
   → Check network connectivity

2. "IMEI not found"
   → Insert test data: psql $DATABASE_URL < test-data.sql
   → Check tenant_id matches
   → Verify IMEI spelling

3. "409 Conflict on checkout"
   → Normal! Device sold during checkout
   → Show user: "Device sold, rescanning..."
   → Retry with different IMEI

4. "Dev server won't start"
   → Check port 3000 available
   → Kill existing process: kill -9 $(lsof -ti:3000)
   → Restart with: pnpm dev
